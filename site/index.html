The Problem

Unstructured logs are an engineering tax
disguised as a logging problem.

A JWT expires in auth-service (Go), cascades through doc-service (Java), and triggers a retry loop in notification-service (Node.js). By the time an engineer investigates, they're staring at 40,000 lines of unstructured text across three Kibana tabs. Manually.

🧵

Three languages, three log shapes

Go uses zap.Error(), Java uses log.error(), Node uses winston.error(). Each team following its own conventions means no machine can correlate them — even if the data is all there.

↑ Manual correlation across every incident
🎯

Errors as strings, not types

logger.error("payment failed: " + err.Error()) tells a human something happened. It tells an AI nothing. You can't route, classify, or deduplicate a free-form string. Every call to the LLM is a cold start.

↑ LLM triage hallucination on bad input
📉

No dropped-message detection

When a Kafka consumer crashes mid-trace, logs are silently lost. Without a sequence counter per service per request, you have no way to know whether a gap in your trace is silence or a missing log.

↑ Root causes hiding behind silent gaps
⏱️

MTTR measured in hours

Google SRE data: the median P1 incident takes 2–4 hours to resolve in systems with unstructured logging. Structured contracts with AI triage push this under 8 minutes for known failure classes.

↑ Engineering hours lost per incident
⚖️

PII in logs = compliance risk

Without enforced contracts, developers accidentally log email, token, and ssn fields. The logscanner CI rule catches these before they merge — not after the audit.

↑ GDPR / HIPAA exposure at every deploy
🔀

AI can't triage what it can't parse

LLMs are powerful, but they hallucinate when given inconsistent input. ObservaLog guarantees the AI receives typed Err structs, deterministic event names, and cross-language sequence counters — every time.

↑ Triage suggestions you can't trust

The wire format — what every log emits
Part A — 55-byte fixed-position header (read at hard-coded byte offsets, zero JSON parsing)
1|trc_7f2a1b9e4d01|spn_004|spn_001|04|2|3|2|1748268153812
schema v1 (byte 0)
trace_id (2–15)
span_id (17–23)
parent_span (25–31)
seq (33–34)
service code (36)
level (38)
outcome (40)
ts_ms unix (42–54)
Part B — abbreviated JSON (event→e, duration_ms→ms, error→er …)
{"e":"provider.send.rejected","m":"Provider rejected document send","er":{"ek":"RateLimitExceeded","ec":"PROVIDER_QUOTA_EXCEEDED","rt":true},"c":{"di":"doc123","hs":429},"o":"failure","ms":87}

After GZIP at the Fluent Bit layer, the size difference vs. a binary protocol is under 10 bytes per entry — with universal jq / Kibana / grep compatibility.

// Decision point — outcome + duration required (scanner enforces this)
func handlePermissionCheck(ctx context.Context, docID string) error {
    start := time.Now()

    ok, err := checkPermission(ctx, docID)
    if err != nil {
        log.Error(ctx, "auth.permission.rejected", "Permission check failed", log.F{
            "doc_id":      docID,
            "outcome":     log.Failure,
            "duration_ms": time.Since(start).Milliseconds(),
            "error": &log.Err{
                Kind: "PermissionDenied", Code: "AUTH_PERM_DENIED",
                Message: err.Error(), Retryable: false,
            },
        })
        return err
    }

    log.Info(ctx, "auth.permission.checked", "Permission granted", log.F{
        "doc_id":      docID,
        "outcome":     log.Success,
        "duration_ms": time.Since(start).Milliseconds(),
    })
    return nil
}

CI Enforcement

Bad logs never
reach production.

The logscanner Rust binary runs in your CI pipeline on every pull request. Five rules. Three languages. Violations exit 1 and block merge.

AI Intelligence

95% of LLM calls
eliminated by design.

The brain fingerprints every error with xxHash64 at emit time (~8ns). Known errors return a cached RepairId instantly. Only genuinely new failure patterns reach the LLM — and they arrive with the exact code that was running at failure time.

Fingerprint deduplication

xxHash64(service_byte | event | error_code | ctx_primary_key). Identical algorithm across Go, Java, Node.js, and Rust — verified by cross-language parity tests. A BIGINT in log_index with a partial HASH index. Cache hit → skip LLM. That's 95% of production incidents.

Failure classification before LLM

The brain classifies Isolated · Cascading · ExternalDependency · Timeout · GapDetected · Partial entirely from the typed columns in log_index — no payload read, no LLM. 95% of triage decisions are a single O(1) TimescaleDB point lookup.

Typed RepairId output

The LLM is constrained to ten typed outputs. Not free-form prose. A RATE_LIMIT_BACKOFF repair can trigger an automated configuration update. An UNKNOWN escalates to a human.

NETWORK_RETRY RATE_LIMIT_BACKOFF VALIDATION_FIX DATABASE_FIX AUTH_REFRESH DEPENDENCY_FALLBACK CONFIG_FIX RESOURCE_EXHAUSTED CONSISTENCY_FIX UNKNOWN

Version-pinned code context

Before calling the LLM, the brain fetches the exact code that was running at failure time — using the version log field (git SHA) to index source via Semble (Model2Vec + BM25 + RRF). A fix deployed yesterday isn't a valid triage target for a failure that happened yesterday.

Installation

Three libraries.
One wire format.

All libraries are available via GitHub Packages. Trigger a release tag to publish. The Go library is available directly via the Go module proxy with no registry configuration.

Go

Install

go get github.com/darshanredkar11/observalog/observalog-go@latest

No registry setup needed. Go fetches directly from GitHub.
